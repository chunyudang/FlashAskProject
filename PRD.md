## 1. 产品概述

### 1.1 产品简介

「闪问(FlashAsk)」是一款以游戏化闯关为核心机制的移动端问答应用。通过编辑团队精心策划的通识知识题目（涵盖科学、社会、逻辑、生活、人文等领域），引导用户在答题过程中提升理性思维与综合认知能力。产品名「闪问」寓意"闪电般快速问答"，英文名 FlashAsk 呼应"闪问"的快速、轻量特性，致力于利用"5分钟碎片时间"为用户提供高效的思维训练体验。

### 1.2 产品愿景

让知识积累变成一种令人上瘾的游戏体验，让用户在每天 5 分钟的碎片时间里，获得有收获的学习体验。

### 1.3 产品 Slogan

5 分钟，给大脑充个电；不背答案，只练思维

## 2. 产品定位与用户分析

### 2.1 目标用户画像

| 用户类型 | 特征描述 | 核心需求 |
|----------|----------|----------|
| **核心用户** | 18-35 岁职场人，通勤/排队/午休等碎片时间较多，有自我提升意愿但缺乏持续学习动力 | 低门槛、有趣、有成就感的知识获取方式 |
| **次核心用户** | 在校学生（高中/大学生），备考之余拓宽知识面 | 系统化知识体系，可量化的成长感 |
| **泛用户** | 对通识知识感兴趣的各年龄段人群 | 娱乐与学习兼顾的内容消费 |

### 2.2 用户痛点

| 痛点 | 解决方案 |
|------|----------|
| 想学习但没动力坚持 | 游戏化机制提供即时正反馈，降低坚持门槛 |
| 碎片时间不知道做什么 | 5 分钟一关，完美匹配碎片时间 |
| 知识面窄但不知从何入手 | 结构化关卡引导逐步拓展认知边界 |
| 传统学习枯燥乏味 | 答题 + 闯关 + 竞技，让学习像玩游戏 |

### 2.3 竞品分析

| 竞品 | 优势 | 劣势 | 我们的机会 |
|------|------|------|------------|
| 头脑王者（微信小程序） | 社交属性强，用户量大 | 题库质量参差，知识体系零散 | 结构化课程 + 高质量题库 |
| 知乎/得到 | 内容深度好 | 缺乏游戏化，门槛高 | 轻量游戏化降低认知负担 |
| 答题类综艺（百万英雄等） | 紧张刺激 | 时效性强，不可持续 | 永久可用内容 + 成长体系 |
| 纯游戏（开心消消乐等） | 游戏性强，上瘾 | 无学习价值 | "有收获的娱乐" |

## 3. 核心玩法设计

### 3.1 核心循环

```
选择关卡 → 答题(固定6题) → 即时反馈(对/错) → 结算页(得分+解析列表) → 分享/下一关
```

### 3.2 关卡通则

| 规则项 | 说明 |
|--------|------|
| 每关题目数 | 固定 6 题（MVP，后续可调整为 5/10/15） |
| 通关条件 | 正确率 ≥ 60%（即答对 ≥ 4 题） |
| 失败处理 | 提示"未通关"，可立即重试，无额外惩罚或复习强制流程 |
| 解锁机制 | 仅线性解锁，无每日挑战、无体力限制 |

### 3.3 计分规则

| 项 | 基础分 | 说明 |
|----|--------|------|
| 答对一题 | +100 分 | 基础得分 |
| 通关奖励 | +200 分 | 完成关卡即可获得 |

## 4. 功能详细说明

### 4.1 MVP（P0）功能

#### 4.1.1 用户系统

| 功能 | 描述 |
|------|------|
| 手机号登录 | 短信验证码登录，首次登录自动注册 |
| 个人资料 | 昵称、头像（默认生成）、总积分、关卡进度 |

#### 4.1.2 首页 — 关卡地图

- **学科选择入口**：5 大学科以卡片形式横向排列，显示学科名称、完成进度（如 3/5 关）
- **关卡列表视图**：当前学科的关卡片列表，显示关卡序号、名称、通关状态（已通关/未通关）、锁定状态
- **关卡锁定规则**：必须通关前一关才能解锁下一关；第一关始终开放
- **视觉反馈**：已完成关卡有通关标识、当前可玩关卡高亮、未解锁关卡有锁定图标

#### 4.1.3 答题核心流程

- **答题前**：显示关卡名称、题数（6 题），有"开始答题"按钮
- **答题中**：
  - 顶部：关卡进度条（1/6）、当前得分
  - 中部：题目文字
  - 下部：4 个选项按钮（A/B/C/D）
  - 选择后：选中态高亮 → 即时反馈对错（绿色/红色）→ 自动进入下一题
- **答题结束**：全部 6 题完成后自动进入结果结算页

#### 4.1.4 结果结算页

- **顶部**：通关/未通关标识
- **中部**：
  - 本关得分
  - 答题统计：正确数/总题数、正确率
  - 积分获得明细
  - 全部题目解析列表（每题展示正确答案 + 知识解析）
- **底部按钮**：
  - "再来一次"（始终显示）
  - "下一关"（通关时出现）
  - "返回关卡地图"

#### 4.1.5 个人中心

- 用户头像/昵称
- 总积分
- 总通关数
- 总答题数 / 正确率
- 设置入口

## 5. 关卡与题库设计

### 5.1 学科体系

| 学科 | 描述 | MVP 关卡数 | 每关题数 |
|------|------|-----------|----------|
| 🧬 科学 | 物理、化学、生物、天文、科技前沿 | 5 | 6 |
| 🌍 社会 | 历史、地理、政治、经济、法律常识 | 5 | 6 |
| 🧩 逻辑 | 推理题、数学趣题、模式识别、思维陷阱 | 5 | 6 |
| 🏠 生活 | 健康饮食、安全常识、实用技能、理财基础 | 5 | 6 |
| 📚 人文 | 文学经典、哲学思想、艺术鉴赏、语言文化 | 5 | 6 |
| **合计** | | **25** | **150** |

> 注：每关 6 题，25 关共 150 道用于作答的题。另有备用池，总题库 500+。

### 5.2 题目结构规范

```json
{
  "id": "S001001",
  "category": "science",
  "level": 1,
  "difficulty": "easy",
  "question": "光年是衡量什么的单位？",
  "options": ["时间", "距离", "速度", "亮度"],
  "correctIndex": 1,
  "explanation": "光年是天文上常用的距离单位，指光在真空中一年内传播的距离，约 9.46 万亿公里。常见误区是误以为光年是时间单位。",
  "tags": ["#天文学", "#基础概念"]
}
```

### 5.3 题目质量标准

1. **准确性**：答案必须准确无误，引用可靠来源
2. **解析价值**：解析必须解释"为什么"，而非仅告知答案
3. **难度合理**：每关题目难度递增，首关 ≥ 70% 正确率，尾关 ≈ 50% 正确率
4. **无争议性**：避免政治敏感、宗教争议、未定论的学术问题
5. **多样性**：同一关卡内知识点不重复，覆盖该学科多个子领域
6. **趣味性**：适当加入冷知识、反直觉现象等增加趣味

### 5.4 难度曲线

| 关卡序号 | 目标正确率 | 题目难度配比 |
|----------|-----------|--------------|
| 1-2（入门） | ≥ 75% | 80% 简单 + 20% 中等 |
| 3-4（基础） | ≥ 65% | 60% 简单 + 40% 中等 |
| 5（进阶） | ≥ 55% | 40% 中等 + 40% 简单 + 20% 困难 |

> 每科独立遵循此难度曲线。

## 6. 后台管理系统

### 6.1 概述

后台管理系统供编辑团队使用，实现题目内容和关卡结构的基础管理。与 App 客户端共享同一后端服务，管理端以独立 Web 页面形式呈现。MVP 阶段仅保留编辑最核心的操作能力，运营分析等非必要功能延后迭代。

### 6.2 角色说明

MVP 阶段仅设单一「编辑」角色，所有登录后台的管理员拥有同等操作权限（题目 CRUD + 分类/关卡管理 + 用户查看）。

### 6.3 功能模块

#### 6.3.1 题目管理

| 功能 | 描述 |
|------|------|
| 题目列表 | 表格展示全部题目，支持按学科/关卡筛选，关键词搜索，分页浏览 |
| 新增题目 | 基础表单：题目文本、4 个选项、正确答案、解析、所属学科和关卡 |
| 编辑题目 | 修改已有题目的任意字段 |
| 删除题目 | 软删除（标记删除，保留数据） |

#### 6.3.2 分类与关卡管理

| 功能 | 描述 |
|------|------|
| 学科管理 | 新增/编辑/删除学科（名称、图标） |
| 关卡管理 | 在学科下新增/编辑/删除关卡（序号、名称、题数配置） |

#### 6.3.3 用户查看

| 功能 | 描述 |
|------|------|
| 用户列表 | 查看注册用户基本信息（昵称、手机号、注册时间、总积分、通关数），支持搜索 |

### 6.4 管理端页面清单

| # | 页面 | 说明 |
|---|------|------|
| 1 | 登录页 | 管理员账号密码登录 |
| 2 | 题目列表 | 题目管理与筛选 |
| 3 | 题目编辑/新建 | 题目表单编辑 |
| 4 | 学科管理 | 学科分类配置 |
| 5 | 关卡管理 | 关卡结构与题目配置 |
| 6 | 用户列表 | 用户基础信息查看 |

### 6.5 技术方案

- **前端**：React + Ant Design（独立 Web 后台）
- **后端**：与 App 共享 RESTful API 服务，增加 Admin 路由模块
- **鉴权**：管理员账号密码登录，JWT 鉴权

## 7. 用户流程

### 7.1 主流程

```
启动App
  ├─ 首次使用 → 引导页（3 页：玩法介绍）
  │             └─ 登录/注册
  │                 └─ 手机号+密码登录/注册
  └─ 非首次 → 首页（关卡地图）
                │
                ├─ 选择学科
                │   └─ 选择关卡
                │       ├─ 锁定 → Toast 提示"请先通关前一关"
                │       └─ 开放 → 进入答题准备页
                │                 └─ 开始答题
                │                     ├─ 答题循环（6 题）
                │                     │   ├─ 选择答案
                │                     │   └─ 反馈对错 → 下一题
                │                     └─ 结果结算（得分 + 解析列表）
                │                         ├─ 分享成绩
                │                         ├─ 再来一次
                │                         ├─ 下一关（通关时）
                │                         └─ 返回关卡地图
                │
                └─ 个人中心
```

### 7.2 异常流程

| 场景 | 处理方式 |
|------|----------|
| 网络断开 | 显示离线提示，已加载的关卡可继续作答，完成后暂存，联网同步 |
| 答题中途退出 | 弹窗确认"确定退出？当前进度将丢失" |
| 题目加载失败 | 显示重试按钮，5 秒后自动重试 |

## 8. UI/UX 设计原则

### 8.1 视觉风格

| 维度 | 要求 |
|------|------|
| 风格定位 | 年轻、活力、清爽、知识感 |
| 主色调 | 科技蓝 (#2563EB) + 知识紫 (#7C3AED) 渐变为品牌色 |
| 辅助色 | 正确绿 (#10B981)、错误红 (#EF4444)、警告橙 (#F59E0B) |
| 字体 | 无衬线字体（PingFang / Noto Sans），层级清晰 |
| 图标风格 | 线性图标 + 填充图标结合，统一圆角 |
| 动画风格 | 流畅自然（300ms 标准缓动），不花哨不卡顿 |

### 8.2 交互规范

| 规范 | 说明 |
|------|------|
| 点按区域 | 最小 44×44pt，确保易用性 |
| 页面切换 | 右滑返回，iOS 原生手势优先 |
| 加载状态 | 骨架屏优先，无加载转圈 |
| 空状态 | 有插图 + 文字引导 + 操作按钮 |
| 反馈延迟 | 点击到反馈 ≤ 100ms |

## 9. 技术架构

### 9.1 技术选型

| 层级 | MVP 技术方案 | 说明 |
|------|-------------|------|
| 客户端（用户端） | UniApp | 一套代码编译到 iOS/Android/小程序，降低 MVP 开发成本 |
| 管理端（后台） | React + Ant Design | 独立 Web 后台，供编辑团队使用 |
| 本地存储 | SQLite | 题目缓存 + 用户进度本地持久化 |
| 后端框架 | 轻量级 RESTful API（Node.js Koa） | 用户端与管理端共享同一后端服务，管理端增加 Admin 路由模块 |
| 数据库 | SQLite（MVP） | MVP 阶段使用 SQLite 快速开发和部署；后续可根据用户量平滑迁移至 PostgreSQL 等大型数据库 |
| 管理端鉴权 | JWT + 角色权限中间件 | 独立的登录鉴权系统，支持角色级权限控制 |
| 云服务 | 阿里云 / 腾讯云 | 国内低延迟部署 |

> 注：MVP 阶段不引入缓存、消息推送、CI/CD 等基础设施，保留至后期版本按需迭代。

## 10. 数据模型

### 10.1 核心实体关系

```
User 1 ── N UserProgress
User 1 ── N UserAnswer

Category 1 ── N Level
Level 1 ── N Question

UserProgress N ── 1 Level
UserAnswer N ── 1 Question
```

### 10.2 数据表定义

#### 10.2.1 users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| nickname | VARCHAR(50) | 昵称 |
| avatar_url | VARCHAR(255) | 头像 URL |
| phone | VARCHAR(20) | 手机号 |
| password_hash | VARCHAR(255) | bcrypt 密码哈希 |
| token | VARCHAR(100) | 自动登录凭证 |
| total_score | INT | 总积分 |
| total_correct | INT | 累计正确题数 |
| total_wrong | INT | 累计错误题数 |
| last_login_at | TIMESTAMP | 最后登录时间 |
| created_at | TIMESTAMP | 注册时间 |

#### 10.2.2 categories（学科分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | VARCHAR(50) | 学科名称 |
| icon | VARCHAR(50) | 图标标识 |
| sort_order | INT | 排序 |
| description | TEXT | 学科描述 |

#### 10.2.3 levels（关卡表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| category_id | UUID | 关联学科 |
| name | VARCHAR(100) | 关卡名称 |
| level_number | INT | 关卡序号（1-5） |
| pass_threshold | INT | 通关所需正确数 |
| description | TEXT | 关卡描述 |
| created_at | TIMESTAMP | 创建时间 |

#### 10.2.4 questions（题目表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| level_id | UUID | 关联关卡 |
| question | TEXT | 题目内容 |
| options | JSONB | 选项数组（4 项） |
| correct_index | INT | 正确答案索引（0-3） |
| explanation | TEXT | 解析内容 |
| tags | JSONB | 标签数组 |
| difficulty | VARCHAR(10) | 难度（easy/medium/hard） |
| sort_order | INT | 关卡内排序 |

#### 10.2.5 user_progress（用户进度表）

| 字段 | 类型 | 说明 |

## 11. 验收标准

### 11.1 功能验收

| # | 验收项 | 标准 |

| 1 | 用户注册/登录 | 手机号 + 微信两种方式均可用 |

| 7 | 刷新状态 | 退出重进后进度不丢失 |
| 8 | 网络异常 | 离线状态下正常提示 |
| 9 | 后台题目管理 | 题目新增/编辑/删除功能可用，筛选搜索正常 |

| 11 | 后台用户查看 | 用户列表展示正常，支持搜索 |

## 12. 迭代路线图

### V1.0 MVP（第 1-2 个月）

核心答题体验闭环 + 基础管理后台：
- 用户系统（登录/注册）
- 5 大学科 × 5 关卡 = 25 关，500+ 题
- 答题流程（固定 6 题/关）
- 即时对错反馈 + 结算页解析列表
- 积分系统（答对+100，通关+200）
- 个人中心 + 关卡进度
- 基础 UI/UX（UniApp + RESTful API + SQLite）
- 后台管理系统（题目 CRUD + 学科/关卡管理 + 用户查看）

### V1.1 激励增强（第 3 个月）

提升留存与活跃：
- 每日挑战
- 连击 Combo + 声效
- 成就系统
- 排行榜（好友/全局）
- 每日签到
- 每题限时（15 秒）

### V1.2 商业化探索（第 4 个月）

建立变现基础：
- 道具系统（跳过/排除/加时）
- 激励视频广告接入
- 皮肤/主题商店
- 错题本
- 社交分享

### V2.0 社区化（第 5-6 个月）

构建生态：
- 用户投稿题目（审核机制）
- 对战模式（异步匹配）
- 知识图谱与薄弱点分析
- 自定义头像框/称号
